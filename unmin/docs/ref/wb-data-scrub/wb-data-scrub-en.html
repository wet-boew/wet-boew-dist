Data Scrub

Questions or comments?

Purpose

This component provides the ability to delete the Personal Identifiable Information (PII) from a form fields before submit.

Here is the list of the PII that the plugin will search and remove:

  • All the combinations of 8 digits and more
  • Canadian serial passport number
  • Email
  • Canadian postal code
  • Any potential user name values added in this format username=theUserValue, username:theUserValue, user=theUserValue, user:theUserValue
  • Any potential password values added in this format password=thePassValue, password: thePassValue pass=thePassValue, pass:thePassValue

For more details about the scrubbing PII behaviour please check on the wet-boew core helper wb.findPotentialPII

Use when

  • forms might contain Personal Identifiable Information (PII), this plugin will identify and delete this information before form submit.

Do not use when

  • a form is using Form validation plugin
  • a form is using Postback plugin

How to implement

  1. Add the following data attribute data-rule-wbscrub for each form element that might contain personal information. 
    <input id="email" name="email" type="email" data-rule-wbscrub />
  2. Optionally, if you want the form to submit a parameter whenever a PII is found and removed, create an hidden input with the attributedata-wbscrub-flag with the name and value of your choice. 
    <input name="scrub" value="1" data-wbscrub-flag />

Events

Document the public events that can be used by implementers or developers.

Event Trigger What it does
wb-init.wb-data-scrub Triggered manually (e.g., $( ".wb-data-scrub" ).trigger( "wb-init.wb-data-scrub" );). Used to manually initialize the wb-data-scrub plugin
wb-ready.wb-data-scrub (v4.0.24+) Triggered automatically after wb-data-scrub initializes. Used to identify the element where steps form has initialized (target of the event) 
$( document ).on( "wb-ready.wb-data-scrub", ".wb-data-scrub", function( event ) {});

$( document ).on( "wb-ready.wb", function( event ) {});
